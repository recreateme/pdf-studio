## 3 架构设计

### 3.1 分层架构

```
┌────────────────────────────────────────────────┐
│               UI 层（app/）                    │
│  pages → widgets → workers                    │
│  （Qt信号/槽驱动，纯展示和交互逻辑）            │
├────────────────────────────────────────────────┤
│             Worker 层（app/workers/）           │
│  BaseWorker → 各具体Worker                    │
│  （QRunnable，在线程池中执行，桥接UI与核心）    │
├────────────────────────────────────────────────┤
│             核心业务层（core/）                │
│  pdf/ · image/ · ocr/ · web/ · export/        │
│  （纯Python逻辑，无Qt依赖，可独立测试）         │
├────────────────────────────────────────────────┤
│             配置与工具层（app/config/ utils/）  │
│  settings · constants · helpers · logger      │
└────────────────────────────────────────────────┘
```

**设计原则**：
- **核心业务层（core/）** 完全不依赖 Qt，可单独测试、复用
- **Worker 层** 负责将同步阻塞操作转移到线程池，保证 UI 不卡顿
- **UI 层（pages/）** 只处理用户交互和结果展示，不包含业务逻辑

### 3.2 信号流

```
用户操作（按钮点击/文件拖入）
    ↓
Page 创建 Worker 并提交到线程池
    ↓
Worker.run_task() 在工作线程执行
    ↓
signals.progress(int, int)    →  更新进度条（Qt跨线程安全）
signals.message(str)          →  更新状态文字
signals.finished(object)      →  任务完成，更新UI、显示Toast
signals.error(str)            →  任务失败，显示错误信息
signals.cancelled()           →  用户取消
```

## 4 核心模块说明

### 4.1 `core/pdf/processor.py`

| 类 | 职责 |
|----|------|
| `PDFReader` | 读取PDF元信息、渲染页面为PNG字节、生成缩略图 |
| `PDFSplitter` | 按范围/页数/书签/大小拆分PDF |
| `PDFMerger` | 合并多个PDF，可选自动书签，支持重复检测 |
| `PDFCompressor` | 重新渲染页面降低DPI实现压缩，支持三种预设 |
| `PDFEncryptor` | AES-256加密/解密，支持打印/复制/编辑权限控制 |
| `PDFWatermarker` | 添加文字水印（透明度、旋转、位置可控） |
| `PDFPageNumberer` | 添加页码（6种位置，自定义格式） |
| `PDFPageEditor` | 页面旋转、删除、裁剪 |

**数据类型**（使用 `@dataclass`）：
- `PDFInfo` - 文档元信息（页数、大小、书签、密码状态）
- `PDFPageInfo` - 单页信息（宽高、旋转、是否有文字）
- `SplitOptions` - 拆分配置
- `MergeOptions` - 合并配置
- `CompressOptions` - 压缩配置
- `WatermarkOptions` - 水印配置
- `PageNumberOptions` - 页码配置

### 4.2 `core/image/` 图像模块

#### `converter.py`

| 类 | 职责 |
|----|------|
| `PDFToImageConverter` | PDF 页面转 PNG/JPEG/WebP 等，支持 DPI、页范围、锐化/灰度/去噪 |
| `ImageToPDFConverter` | 图片列表转 PDF，支持单图单页 / 九宫格布局 |
| `ImageEnhancer` | 扫描增强（纠偏、去黑边、去噪、对比度、二值化等） |

#### `merger.py`

| 类 | 职责 |
|----|------|
| `ImageMerger` | 多图合并：纵向/横向长图、自定义行列网格、分页输出 |
| `ImageMergeOptions` | 外边距/间距、固定画布宽高、WebP 无损、多页后缀模板等 |

合并引擎提供 `merge()`（落盘）与 `render_preview()`（低分辨率预览，供 Worker 异步渲染；UI 当前以单图大图预览为主）。

#### `compressor.py`

| 类 | 职责 |
|----|------|
| `ImageCompressor` | 单张/批量图片压缩 |
| `ImageCompressOptions` | 三种模式：`target_size`（不超过目标字节，二分搜索质量+必要时缩放）、`scale`（按比例缩小）、`quality`（固定质量） |

### 4.5 UI 样式与共用控件

为降低「白底 + 白控件」造成的视觉疲劳，应用层提供两层样式：

| 模块 | 路径 | 说明 |
|------|------|------|
| 全局补充样式 | `app/ui/app_styles.py` | 启动与主题切换时注入 `QApplication`：页面浅灰底、卡片描边、输入框/次要按钮/Tab/分段选择器底色 |

**图片合并列表**（`image_merge_list.py`）：上方 `ListMode` 文件列表，下方选中项大图预览（单击预览、双击系统默认程序打开）。

**拖放区**（`common.py` `DropZone`）：使用 `drop_zone_stylesheet()`，随主题切换浅灰底 + 虚线边框。

## 5 接口规范

### 5.1 Worker 接口（BaseWorker）

```python
class MyWorker(BaseWorker):
    def __init__(self, *args):
        super().__init__()
        # 初始化参数

    def run_task(self) -> Any:
        """
        在工作线程中执行的任务逻辑。

        - 可通过 self.is_cancelled() 检查取消请求
        - 可通过 self.emit_progress(current, total) 发送进度
        - 可通过 self.emit_message(msg) 发送状态消息
        - 返回值将通过 signals.finished 信号传递给UI

        Raises:
            Exception: 任何未捕获异常会触发 signals.error 信号
        """
        for i, item in enumerate(self.items):
            if self.is_cancelled():
                break
            # 处理 item...
            result = process(item)
            self.emit_progress(i + 1, len(self.items))
        return results
```

### 5.2 核心处理函数签名约定

```python
def some_processor(
    input_path: str | Path,
    options: SomeOptions,
    progress_cb: Optional[Callable[[int, int], None]] = None,
) -> Path | list[Path]:
    """
    Args:
        input_path: 输入文件路径
        options: 处理选项（dataclass）
        progress_cb: 可选进度回调，(current, total)

    Returns:
        单个输出路径或路径列表

    Raises:
        FileNotFoundError: 文件不存在
        ValueError: 文件格式错误或参数无效
    """
```

## 6 线程模型

### 6.1 线程池配置

```python
pool = QThreadPool.globalInstance()
pool.setMaxThreadCount(8)          # 默认最大8个工作线程
```

### 6.2 线程安全

- Qt 信号/槽跨线程传递是线程安全的（Qt 自动队列化）
- Worker 中**禁止直接操作任何 Qt UI 控件**
- 所有 UI 更新必须通过信号/槽进行

### 6.3 任务队列

后台任务通过 `submit_worker()` 提交，由 `TaskHub` 统一管理：

| 策略 | 说明 |
|------|------|
| 并发上限 | `workflow.max_workers`，与 `QThreadPool` 一致 |
| 队列上限 | `workflow.queue_max_size`，含「进行中 + 排队中」 |
| 有空闲线程 | 立即启动，状态为 `running` |
| 线程已满但未达队列上限 | 进入 FIFO 排队，状态为 `queued`，右上角 InfoBar 提示 |
| 队列已满 | **拒绝**新任务，Warning 提示；调用方 `submit_worker()` 返回 `False` |

预览类 Worker（`ThumbnailWorker` / `PDFPageRenderWorker` / `ImageMergePreviewWorker`）不进入队列，直接提交线程池。

首页「后台任务」面板可查看进行中 / 排队 / 已完成任务。

### 6.4 取消机制

```python
# 在UI层
worker.request_cancel()    # 设置取消标志

# 在Worker中（run_task内部循环）
for item in items:
    if self.is_cancelled():
        break              # 检测到取消则退出循环
    process(item)
```

## 9 打包部署

### 9.1 PyInstaller（推荐）

```bash
pip install pyinstaller
python scripts/ensure_deps.py --install
pyinstaller pdf_studio.spec

# 输出：dist/PDFStudio/PDFStudio.exe（onedir 目录模式）
```

`pdf_studio.spec` 要点：
- 入口：`main.py`；图标：`app/resources/icons/app.ico`
- 全量收集 `cryptography`；Conda 环境自动附带 expat/ffi 等运行时 DLL
- 隐藏导入覆盖 `app.*` / `core.*`（含图片合并、压缩、全局样式）
- 排除 Playwright / PaddleOCR / pytest 等体积大或开发专用依赖
- Windows 版本资源：`app/resources/version_info.txt`（产品名 pd-studio）

### 9.1.1 打包冒烟（自动化）

```bash
# 完整流程：pytest → PyInstaller → 冻结包 --pack-smoke → GUI 启动探测
python scripts/pack_smoke.py

# 仅验收已有 dist
python scripts/pack_smoke.py --skip-build --skip-pytest --dist-dir dist/PDFStudio
```

冻结包内自检：`PDFStudio.exe --pack-smoke`，结果写入同目录 `pack_smoke_result.txt`。  
`pdf_studio.spec` 已补充 Conda 运行时 DLL、`pypdf` 加密子模块等；`app/utils/deps.py` 对冻结环境有 `crypt_provider` 回退检测。

### 9.2 Nuitka（更小体积，可选）

```bash
pip install nuitka
nuitka --standalone --onefile \
       --enable-plugin=pyqt6 \
       --windows-icon=app/resources/icons/app.ico \
       --output-dir=dist \
       main.py
```

### 9.3 打包注意事项

1. **Playwright Chromium**：需手动将 `~/.cache/ms-playwright` 目录打包进发布包，或在首次运行时触发下载
2. **OCR 模型**：RapidOCR 模型在首次运行时自动下载；可预先下载放入 `~/.rapidocr/` 实现完全离线
3. **中文字体**：Windows 打包时建议包含 Microsoft YaHei 字体，避免中文显示问题
4. **macOS 代码签名**：分发给其他用户时需要签名，否则触发安全警告

### 10.2 新增核心处理器

4. 编写 `tests/` 下的单元测试

### 10.3 新增 Worker

在 `app/workers/base_worker.py` 中注册具体 Worker，例如：

| Worker | 对应核心模块 |
|--------|----------------|
| `ImageMergeWorker` | `core.image.merger.ImageMerger` |
| `ImageCompressWorker` | `core.image.compressor.ImageCompressor` |
| `ImageEnhanceWorker` | `core.image.converter.ImageEnhancer` |
| `PDFCompressWorker` | `core.pdf.processor.PDFCompressor` |

```python
class MyTaskWorker(BaseWorker):
    def __init__(self, input_path: str, options: MyOptions):
        super().__init__()
        self.input_path = input_path
        self.options = options

    def run_task(self):
        from core.my_module.processor import MyProcessor
        processor = MyProcessor()
        return processor.process(
            self.input_path,
            self.options,
            progress_cb=self.emit_progress,
        )
```

## 12 已知限制与后续规划

### 12.1 产品定位

PDF Studio（pd-studio）定位为 **WPS 免费版 PDF 工具箱补位**：在 WPS 中阅读与写文档，在本应用中完成 WPS 会员才提供的 PDF 处理能力（本地、离线、无广告）。

### 12.2 当前已知限制

| 限制 | 影响 | 说明 |
|------|------|------|
| 合并时统一页面尺寸效果有限 | 仅调整 CropBox | 复杂版式可能不理想 |
| OCR 无表格结构化 | 表格识别为普通文本 | 后续可集成版面模型 |
| Playwright 批量转换为串行 | 批量网页转换较慢 | 已支持有限并行（设置可调 1–4） |
| 去水印仅适用重复水印 | 复杂水印无法完全去除 | 已在 UI 说明 |
| 打包版不含 Playwright | 网页功能依赖系统 Chrome/Edge | spec 已排除 playwright |
| 图片签名非 CA 证书 | 无法律效力数字签章 | 仅视觉标记 |
| 任务队列满时拒绝新任务 | 需等待或调高队列上限 | 首页可查看排队状态 |
| 图片目标大小压缩 | 极小目标可能需大幅缩小分辨率 | PNG 目标模式自动转 JPEG |
| 图片合并动图 | 仅使用 GIF 首帧 | 文档已说明 |
